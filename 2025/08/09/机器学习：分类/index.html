机器学习：分类

阈值和混淆矩阵

分类阈值

定义：在二元分类中，一个介于0和1之间的数字。用于将逻辑回归模型的原始输出转换为正类别或负类别的预测结果。该值是人为选择的值。

通过以下操作可以得到预测类别：

  • 逻辑回归模型输出一个介于0到1之间的原始值；

  • 如果原始值大于分类阈值，则预测为正类别；

  • 如果原始值小于分类阈值，则预测为负类别；

  • 对于相等的情况，工具/框架不同，处理的方式也不相同，例：Keras库会预测为负类别。

除此之外，分类阈值不一定要是”中间值”，有时候，一种错误的代价(假正例和假负例之间)大于另一种，那么这时候，应该尽量把分类阈值向避免代价大的错误修改。

混淆矩阵

定义：一种N * N的表格。用于总结分类模型做出的正确和错误预测的数量。

实际正例 实际负例
预测为正例 真正例(TP)：预测的结果和标签一样为正例 假正例(FP)：预测的结果为正例，但标签为负例
预测为负例 假负例(FN)：预测的结果为负例，但标签为正例 真负例(TN)：预测的结果和标签一样为负例

如果实际正例的总数与实际负例的总数不接近，则表示数据集不平衡。

指标

指标均基于单个固定阈值计算得出，并且会随阈值的变化而变化。通常，用户会调整阈值以优化其中一个指标。

准确率

定义：正确的分类预测数量除以预测总数。数学定义为：Accuracy = correct classifications/total classifications = (TP + TN)/(TP + TN + FP + FN)

  • 由于准确率包含混淆矩阵中的所有四种结果（TP、FP、TN、FN），因此在类别数量相近且平衡的数据集的情况下，准确率可以作为衡量模型质量的粗略指标。因此，它通常是执行通用或未指定任务的通用或未指定模型使用的默认评估指标。

  • 完美模型没有假正例和假负例，因此准确率为1.0，即100%。

  • 如果数据集不平衡，或者一种错误（假负例或假正例）的代价高于另一种错误时，此指标不适用。

  • 对于严重不均衡的数据集（其中一个类别出现的频率非常低，例如 1%），如果模型 100% 都预测为负类别，则其准确性得分为 99%，尽管该模型毫无用处。

召回率或真正例率

定义：真正例率(TPR)，即所有实际正例被正确分类为正例的比例，也称为召回率。数学定义为：

Recall(Or TPR) = correctly classified actual positives/all actual positives = TP/(TP + FN)

  • 召回率的另一个名称是检测概率：可以用于回答“此模型检测到正例的比例是多少？”

  • 假设一个完美的模型不会出现假负例，因此其召回率 (TPR) 为 1.0，也就是说，检测率为 100%。

  • 在实际正例数量非常少的不平衡数据集中，召回率比准确率更有意义，因为它衡量的是模型正确识别所有正例实例的能力。对于疾病预测等应用，正确识别阳性病例至关重要。假负例通常比假正例的后果更严重。

假正例率

定义：模型错误地将实际负例预测为正例的比例也称为误报概率。数学定义为：

FPR = incorrectly classified actual negatives/all actual negatives = FP/(FP + TN)

  • 完美的模型不会产生假正例，因此其假正例率为 0.0，也就是说，假正例率为 0%。

  • 在实际负例数量非常少的不平衡数据集中，FPR 作为一个指标就没有那么有意义和实用。

精确率

定义：指模型所有正类别分类中实际为正类别的分类所占的比例。数学定义为：

Precision = correctly classified actual positives/everything classified as positive = TP/(TP + FP)

  • 假设有一个完美的模型则其假正例数为零，因此精确率为 1.0。

  • 在实际正例数量非常少的不平衡数据集中，精确率作为指标的意义和实用性较低。

随着假正例的减少，精确率会提高；随着假负例的减少，召回率会提高。提高分类阈值往往会减少假正例的数量并增加假负例的数量，而降低阈值则会产生相反的效果。因此，精确率和召回率通常呈现反向关系，提高其中一个会降低另一个。

指标的选择

指标 指南
准确率 作为平衡数据集的模型训练进度/收敛情况的粗略指标。
对于模型效果，请仅与其他指标搭配使用。
避免使用不平衡的数据集。
召回率(真正例率) 当假负例的代价高于假正例时使用。
假正例率(误报概率) 当假正例的代价高于假负例时使用。
精确率 当正例预测的准确性非常重要时，请使用此方法。

F1得分

定义：是精确率和召回率的调和平均数。数学定义为：

F1 = 2 * precision * recall/(precision + recall) = 2TP/(2TP + FP + FN)

此指标可平衡精确率和召回率的重要性，对于类别不平衡的数据集，优先于准确率。

  • 当精确率和召回率均为 1.0 的满分时，F1 得分也会为 1.0 的满分。

  • 当精确率和召回率的值接近时，F1 得分也会接近它们的值。

  • 当精确率和召回率相差很大时，F1 将与较差的指标相似。

接收器操作特征曲线(ROC)

定义：在二元分类中，针对不同的分类阈值，绘制真正例率和假正例率的对比图。ROC曲线的形状表明了二元分类模型区分正类别和负类别的能力。

完美的模型在某个阈值下的 TPR 为 1.0，FPR 为 0.0，如果忽略所有其他阈值，则可以用 (0, 1) 点表示(x轴为FPR，y轴是TPR)。

ROC曲线下面积(AUC)

定义：一个介于0.0和1.0之间的数字，表示二元分类模型区分正类别和负类别的能力。

更加正式的定义：AUC是指分类模型认为随机选择的正例确实为正的概率比随机选择的负例为正的概率更高。

  • AUC越接近1.0，模型区分不同类别的能力就越好。

  • 对于二元分类器，如果模型的效果与随机猜测或抛硬币的效果完全一样，则其ROC 曲线为从 (0,0) 到 (1,1) 的对角线，AUC 为 0.5，表示正确对随机正例和负例进行排名的概率为 50%。

精确率与召回率曲线(PRC)

对于不均衡的数据集，PRC和其曲线下面积比ROC和AUC更能直观地比较出模型性能。创建方法：x轴上绘制所有阈值下的召回率，y轴上绘制精确率。

选择模型和阈值

  1. 曲线下面积较大的模型通常是更好的模型，即AUC较大的模型。

  2. ROC曲线上最接近(0,1)的店表示给定模型效果最佳的阈值范围。

  3. 如果假正例的代价高，那么可以选择FPR较低的阈值；相反，如果假负例代价高，那么应该选择TPR较高的阈值；若代价大致相同时，选择TPR和FPR之间可能的最佳平衡。

预测偏差

和线性回归一样，分类模型也有偏差。预测偏差指的是模型预测结果的平均值与数据中真实标签的平均值之间的差异。

如果一个模型是基于包含 5% 邮件为垃圾邮件的数据集训练出来的，那么该模型平均而言应预测其分类出的邮件中有 5% 是垃圾邮件。换句话说，真实标签数据集中的标签平均值为 0.05，而模型的预测结果平均值也应为 0.05。如果情况如此，那么该模型就没有预测偏差。

多类别分类

二元分类的扩展，适用于两个以上的类别。

如果每个示例只能分配到一个类，则可以将分类问题处理为二元分类问题，其中一个类包含多个类中的其中一个，另一个类包含所有其他类。然后，可以对每个原始类别重复此过程。
